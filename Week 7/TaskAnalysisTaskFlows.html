The Brief

Task flows are a tool to help us think through the design before a feature is actually developed. They allow us to interject the user into the flow of the application and determine if the conceptual model agrees with the user model.

  • The system presents the user with paths.
  • The user moves along these paths through actions.

Elements of a Good Task Flow

  • Assumptions about the user or initial conditions
  • User Objective
  • Key/Legend:
    • diamond: user interaction
    • rectangle: page
    • parallelogram: process
  • Show the screen changes
  • Provide ways for the user to recover from error (if applicable)

Different Types of Flows

  • Flowchart
  • IA Diagram
  • State Diagram
  • Wireflow

Tips

  • Remember who your users are, and their technical competency
  • Sketch out the flow in lo-fidelity with your team members first
  • Work out the high-level flow before working towards a mid-to-high fidelity flow
  • Identify different places where the user can get stuck or encounter errors
  • Keep the heuristics for usability in mind!
  • Work in Axure last
